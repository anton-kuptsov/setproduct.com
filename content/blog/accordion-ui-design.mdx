## What is Accordion?

**_Accordion_** _(aka_ [Expansion panel](https://material.angular.io/components/expansion/overview)_)_ — vertically stacked list of options that can expand/collapse to reveal or hide more associated content.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-1.jpg)

## Anatomy

### **States**

Default states for _Accordion_ are:  

-   collapsed
-   expanded
-   onhovered
-   focused
-   disabled

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-2.jpg)

### **Chevrons placement**

Chevron icon represents the opportunity to expand/collapse _Accordion_ item. Once _Accordion_ was clicked, the chevron rotates to indicate the successful expansion. Further collapse returns the chevron to a previous position.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-3.jpg)

Instead of a chevron, you may use the following symbols:  

-   Up / down caret
-   Plus / Minus
-   Up / down arrow

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-4.jpg)

### **Additional icon**

In the case of right-sided _Accordion_, you may place additional icons to strengthen the meaning. Use wisely not to overload the component.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-5.jpg)

In some cases, you may apply colors to visually separate the items:

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-6.jpg)

## Styles & Themes

### **Expanded item**

Highlight expanded state to let a user recognize what is opened in case of multiple _Accordions_. There are several possible styles for the highlighting:  

-   Elevation shadow
-   Background fill
-   Outline stroke

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-7.jpg)

While the elevation works well on both smooth grey and white surfaces, you may fill the _Accordion_ expanded item's background with 8–12% opacity fill. Fits good for white surfaces. Add outline stroke, if necessary.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-8.jpg)

### **Contained Accordion**

Use contained _Accordion_ for most of the cases. Separate its items by a slightly visible divider. This placement either saves some vertical space.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-9.jpg)

### **Separated Accordion**

There are some cases when you can separate _Accordions_ with whitespace. Use this method to divide when it's needless to save vertical space.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-10.jpg)

## Accordion UX & Usability

### **Subtitle to pre-know**

Enable a subtitle to inform your users about what to expect when _Accordion_ expands. This either helps to display the user's pick when everything is collapsed.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-11.jpg)

### **Badge to pre-know**

Either use a badge to indicate the selection hastily, while _Accordion_ is collapsed.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-12.jpg)

### **Helper text**

Enable a helper text for _Accordion_ to ergonomically show up a user's pick when collapsed, especially for mobile viewports.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-13.jpg)

## Mobile patterns

### **Full-width usage**

On mobile devices use only full-width _Accordions_.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-14.jpg)

### **Side menu**

Implement _Accordions_ into the navigation drawer to reveal submenu items.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-15.jpg)

### **Filters**

Use side panels with _Accordions_ to utilize multiple filters, properties, etc.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-16.jpg)

## Desktop patterns

### **Side menu**

Analogous as mobiles, you may implement _Accordion_\-driven side navigation for desktop apps, dashboards, etc.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-17.jpg)

### **Filters**

Depending on the desktop layout structure, _Accordions_ (e.g. as Filters) could be placed as both left or right-sided.

![Accordion UI design tips](/blog/assets/accordion-ui-design/img-18.jpg)
